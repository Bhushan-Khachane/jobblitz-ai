# JobBlitz-AI Build Log

## Mission 1 — Wire the Real Orchestration Loop
**Status: COMPLETE**

Added `orchestration_checkpoints` table to `packages/db/schema.ts` with fields: id, application_id, user_id, graph_state, status, created_at, updated_at, expires_at. Indexed on application_id, user_id+status, and expires_at for efficient querying. Refactored `apps/worker-orchestrator/src/index.ts` to use a dedicated BullMQ queue `"orchestration-jobs"` with concurrency 3 (up from 2). The worker now writes explicit checkpoints to the new table on `GraphInterrupt` with status `"awaiting_approval"` and `expires_at = now() + 24h`. On completion, it updates the `applications` table to `"submitted"` and publishes a Redis notification to `jobblitz:notifications:{userId}`. On failure after retries, it writes `"failed"` to both the applications table and checkpoints table, then publishes a failure event. Added graceful shutdown via `SIGINT`/`SIGTERM` handlers that close the worker, queue, and Redis connections. Modified `packages/agents/src/graphs/application.ts` `executeApplicationNode` to delegate browser automation via HTTP POST to `BROWSER_WORKER_URL/apply` instead of running Stagehand inline — this decouples the orchestrator from browser dependencies. Added `persistResultNode` HTTP PATCH to `LEGACY_API_URL/applications/{id}/status` for backward compatibility with the Python backend. All 36 packages pass `bun run typecheck` with 0 errors.

## Mission 3 — packages/security: Real Implementation
**Status: COMPLETE**

Created `packages/security/src/vault.ts` with `encrypt(plaintext, userId)` and `decrypt(ciphertext, userId)` using PBKDF2(userId + VAULT_MASTER_KEY, randomSalt, 100000, 32, sha256) for key derivation, followed by AES-256-GCM. The `VAULT_MASTER_KEY` is read exclusively from environment variables; no hardcoded fallback exists. Created `packages/security/src/credential-proxy.ts` with a `CredentialProxy` singleton: `put()` generates a UUID token with 120-second TTL, `get()` returns credentials and marks them used (single-use), and a `startEvictionLoop()` setInterval evicts expired entries every 30 seconds. Created `packages/security/src/pii-redactor.ts` with two modes: `redactForLogs()` redacts email, Indian phone numbers, Aadhaar (12 digits), PAN (ABCDE1234F), and credentials after "password:", "token:", or "Bearer "; `redactForLLM()` is stricter and also redacts Title Case 2–3 word sequences (likely full names). Rewrote `packages/security/src/rate-limiter.ts` with `applicationRateLimit(userId, getDailyLimit, getTodayCount)` (callback-based to avoid circular deps with `@jobblitz/db`) and `llmRateLimit(userId, redis)` using Redis ZADD/ZCOUNT with a 1-hour sliding window and max 100 calls. Kept the legacy in-memory `RateLimiter` class for HTTP middleware. Updated `packages/security/src/index.ts` to export all new modules with selective exports to avoid duplicate `redactObject`. Added 4 test files in `packages/security/src/__tests__/` covering vault round-trip, wrong-user decryption failure, credential proxy TTL and single-use behavior, PII redaction for all patterns, and rate-limiter application/LLM sliding window limits. All 13 tests pass, 0 failures. All 36 packages pass `bun run typecheck` with 0 errors.

## Mission 4 — Real pgvector Semantic Search
**Status: COMPLETE**

Added `job_embeddings` and `user_skill_embeddings` tables to `packages/db/schema.ts`, each with a `vector(1536)` column and HNSW index for fast approximate nearest-neighbor search. `job_embeddings` links to `jobs.id` via foreign key with CASCADE delete; `user_skill_embeddings` links to `users.id`. Created `packages/memory/src/embed.ts` with `embedText(text, redis?)` that uses the existing OpenAI/Fake embedding client and caches vectors in Redis with a 7-day TTL (key = `embed:{sha256(text)}`). Created `packages/memory/src/search.ts` with `hybridJobSearch(db, redis, options)` which embeds the query text, performs semantic search against `job_embeddings` using Drizzle SQL with cosine distance, fetches legacy match scores via optional HTTP GET to `LEGACY_API_URL`, and returns a combined ranking weighted 0.6 semantic + 0.4 match. Created `packages/memory/src/indexer.ts` with `indexJob(jobId, jdText, db, redis)` (strips HTML, truncates to 8k chars, embeds, stores) and `indexUserProfile(userId, profile, db, redis)` (concatenates headline/summary/skills, embeds, stores in `user_skill_embeddings`). Updated `packages/memory/src/index.ts` to export all new modules. Updated `upsertJobEmbedding` in `packages/memory/src/jobs.ts` to also populate `job_embeddings` so the generic embeddings table and dedicated table stay in sync. Created `apps/api/src/routers/memory.ts` with three authenticated endpoints: `POST /api/memory/index-job`, `POST /api/memory/index-profile`, and `GET /api/memory/search?q=...&limit=...`. Mounted the router in `apps/api/src/index.ts` at `/api/memory`. All 36 packages pass `bun run typecheck` with 0 errors.
